# Novel approach: Self-Factorized Skill Curriculum (SFSC) — turning a heterogeneous task into the regime the ratchet provably wins

*Not from the SQL literature. Invented from OUR capstone finding. The point (per the operator): don't
restrict to proven approaches — solve the problem.*

## The problem, restated as a lever (not a wall)
Our capstone: the ratchet generalizes for **one skill × many instances** (F1: train operands 1-8 →
held-out 9-12), but NOT for a **heterogeneous task × few instances each** (real SQL: every question a
different query). The published fixes (CoT-distill, RL, decompose-at-inference, synthesize-916K-external-
data) all either need scale/RL compute or do per-question decomposition WITHOUT learning. None turns the
capstone into a *generative* fix.

**The novel idea:** if the ratchet wins at the skill grain, then **make the heterogeneous task skill-
grained — autonomously.** The being factorizes the task into sub-skills *it discovers from its own
verified successes*, manufactures *its own targeted practice* for each (verifier-gated), internalizes each
homogeneously, and composes. Heterogeneous-task → self-discovered set of homogeneous skills → the regime
the ratchet provably wins.

## The mechanism (4 steps, all local, verifier-gated, zero-salary)
1. **Seed.** Solve what it can on the real task (sample-N, free verifier keeps the correct) → a pool of
   verified (problem, solution) traces. (Where v3 stopped — and starved.)
2. **Factorize (self-discovered, the invention step).** The model inspects its verified solutions and
   names the recurring SUB-SKILLS it used — e.g. for SQL: `join-on-FK`, `filter-count`, `group-aggregate`,
   `subquery-compare`, `date-range`. The taxonomy is generated by the model from its own successes (not a
   human/DIN-SQL fixed schema) — verified later by whether practicing them helps.
3. **Self-generate targeted practice (the homogenizing step — the key novelty).** For each discovered
   sub-skill, the model SYNTHESIZES MANY NEW instances exercising *only that sub-skill* on the available
   schemas (new questions + their SQL), each EXECUTION-VERIFIED. This manufactures the "many instances per
   skill" the ratchet needs — from the task's own structure, with no external dataset and no salary. A
   heterogeneous task with few examples becomes K homogeneous sub-skills with many self-made examples each.
4. **Internalize + compose.** Distill each sub-skill homogeneously (the regime that WORKS), and distill
   composition traces (sub-skills chained into full solutions — our F9 recursion). Held-out *full* tasks
   should now lift, because their sub-skills were each internalized at the grain the ratchet generalizes.

## Why this is genuinely novel (vs the field)
- **DIN-SQL** decomposes each question at *inference*, in-context, no learning, fixed scheme. SFSC
  *discovers* the factorization from the model's own successes and *internalizes* the sub-skills into
  weights.
- **SynSQL / synthetic-data** uses externally-authored templates. SFSC's practice is *self-generated,
  self-targeted at the model's own weak/used sub-skills, verifier-gated* — the model authors its own
  curriculum.
- **RL (CogniSQL)** reinforces whole-task reward. SFSC restructures the *learning signal's grain* to match
  where internalization provably transfers (our capstone) — a different axis (representation, not reward).
- It operationalizes a *measured* finding (skill-grain generalization) as a *generative* algorithm.

## Falsifiable test (run after v4, or instead if v4 is flat)
On reachable BIRD (simple+moderate): (a) seed verified solutions; (b) model proposes ~5 sub-skill tags;
(c) for each, self-generate + execution-verify ~20 targeted practice queries (homogeneous); (d) distill
sub-skills + composition; (e) eval held-out FULL questions before→after. NOVEL ✓ iff held-out lifts
where v3 (whole-task, few traces) was flat — i.e. self-homogenization cracks heterogeneous generalization.
Honest kill: if self-generated practice can't be verified (the 8B can't author correct targeted queries)
or composition doesn't transfer — record straight; that bounds *which* heterogeneous tasks self-factorize.

## Risks
- Self-generated practice may be low-yield (8B authors wrong queries) → mitigate: verifier gate + sample-N;
  if still low, the sub-skill is beyond-reach (the deeper-boundary regime, honestly bounded).
- Factorization may be degenerate (model names vague tags) → mitigate: require each tag to have ≥N
  verified distinct instances or drop it.
- This is the boldest arm; v4 (proven recipe) is the control. Run both; compare.

## Salary, intrinsic motivation, cloud traces — the hybrid escalation (operator question)
The free loop STARVES exactly on the beyond-reach sub-skills (the 8B can't self-generate correct traces).
That is the textbook case for SALARY by our own principle (salary pushes a frontier iff it buys what the
free loop structurally can't AND amortizes to ~0):
- **Cloud/teacher traces = the direct fix.** A frontier teacher (claude -p) generates the correct
  reasoning+SQL the 8B cannot, the 8B distills it (teacher-bootstrap — proven in F6, and exactly how
  SLM-SQL/CogniSQL bootstrap small models with teacher/synthetic CoT before RL). Buys the yield the free
  loop lacks.
- **Salary-principle gate (amortize or it's a crutch):** justified ONLY if the bootstrapped 8B then needs
  the teacher LESS — measure frontier-dependence decay (teacher-calls-per-new-sub-skill ↓ over rounds). If
  it decays, salary is a one-time ignition that pays to zero; if not, it's runtime dependence (rejected).
- **Intrinsic motivation = the salary ROUTER (efficiency).** The free verifier already tells the being
  which sub-skills/questions it FAILS — that uncertainty/failure signal is an intrinsic-motivation drive:
  spend salary ONLY there (ask the teacher for exactly the sub-skills the being can't self-solve; let the
  free ratchet handle the reachable ones). (Agent0 uses executor-uncertainty as an RL reward; here it
  *routes the teacher budget* — minimal salary, maximal gap-closing. Economy = learning.)
- **The hybrid (each lever where it belongs):** free SFSC self-practice for REACHABLE sub-skills +
  salary-teacher (uncertainty-routed) for BEYOND-REACH sub-skills → internalize all homogeneously →
  compose. This is how a *small local* model could crack a heterogeneous real task: free where it can,
  salary precisely where it can't, internalized so salary→0.
Test: after a teacher-bootstrap round on the failed sub-skills, does (a) held-out FULL-task accuracy lift,
and (b) the being's SELF-sufficiency rise (it now self-generates traces it previously needed the teacher
for)? (b) is the democratization claim with salary: the teacher graduates itself out.
